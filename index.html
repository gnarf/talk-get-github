GitHub

So I want to tell you how to GitHub, but before I talk about GitHub, let's talk about Git

So what is Git?

  • Revision Control
  • Distributed
  • Speed
  • Data Integrity
  • Non-Linear Workflows
  • Most Popular (source)
Poll: How many people have never used git? have you used other source control? In case you aren't familiar, git is...
→ code goes in a git repository... A history of the codebase, revision, control
→ distributed means every "clone" is it's own local repo (offline anyone)
→ speedy because it's local for most operations
→ because sha1 checksums
→ non-linear... branching and mergeing are super fast, A core assumption in Git is that a change will be merged more often than it is written
→ popular! yay

(dont advance) If you're curious to read more about it, or need some selling points, the wiki page is pretty great

Okay! So what is GitHub?

Build software better, together.
(Emphasis mine)

Another quick poll - how many of you already have a GitHub account? Do you use it for work? For open source?
For those of you that don't know - GitHub - aside from having an adorably cute mascot - is a place for hosting and collaborating on git repositories. Their Motto → really kind of sums it up.

Together...

  • Sharing Knowledge
  • With history for reference
  • Asynchronusly
I want to zero in on the word "together" for a second. What means to me is sharing our knowledge and efforts in an online space, with history for reference, and asynchrounsly. This means you aren't tapping your co-worker on the shoulder and demanding they give you time, and we all get to be more productive.

GitHub

(in case you aren't already convinced)
  • Integrated Issue Tracking
  • Collaborative Code Review (Pull Request)
  • Organizations & Teams
  • Robust API
  • Integrations
  • Free for public
  • Pay for private
  • Pay more for Enterprise
For those of you not using GitHub at work, here's a few selling points in case you want to convice your team.
Building on top of git's powerful tools for managing your code, GitHub adds a bunch of great features.
→ They have an Integrated Issue Tracking system which is something every project needs to solve. You can of course disable it if you already track your issues elsewhere.
→ Collaborative Code Review! .... Pull Requests A.K.A. "PR"s are probably the most important feature offered by GitHub and I'll tell you all about em in a minute or two.
→ github is also Team oriented. you can create organizations, group them by teams, and control access to repos individually.
→ GitHub's website is built using their own API. Almost anything you can do on GitHub can be done with the very well documented API.
→ And because the API is awesome, it integrates with a lot of services. Flowdock, HipChat, Slack, IRC, Jenkins, Travis, JIRA, the list goes on and on.
→ It's free for public repositories, even organizations! If you aren't trying to hide it from the world, you'll never pay a dime. If you pay you unlock private repositories.
→ If you pay even more you can get Enterprise, and run it on your own servers.

Pull Requests

Can't Push? Try a "Pull Request"

Can Push, but want to discuss with team? Also a PR

  • Both general and line comments.
  • Tracked in a branch
  • "Magic links" from comments
    #issue, gh-issue, owner/repo#issue,
    @user, @org/team, sha
  • All done? 👍 (:+1:) Merge it in!

Comments welcome!

  • "Can you explain why we need this?"
  • "Would it be better to...."
  • "I think you need a comment here, it took me a few minutes to figure out"
  • "I tried this code locally, but the tests failed"
Speaking for jQuery at least... Every other open source project I've contributed too was equally happy to have outsiders.
Sometimes what we really need is a fresh pair of eyes to tell us something is greek instead of javascript. Just be friendly and try your best to be helpful to the community.
